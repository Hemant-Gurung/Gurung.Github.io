Dual-Rasterizer

Rasterization is the process of converting a mathematically defined image using points and curves to a picture composed of descrete pixels which is called Raster image. To explain it simply, it is a way to visualize the geometry as pixels on a screen. Rasterization in general is faster beacuse of better convergence when projecting and the computer GPU's are optimized to do such oparation. see source code in Github

Ray tracing

Ray tracing is a method of graphics rendering that simulates the physical behavior of light which includes the reflection, refractions, shadows and indirect ligting. All of the simulation are done using 3D maths and formulas. go to the source code

Shaders

Shader is a piece of code that is executed in Graphics Processing Uni (GPU). It is used to manipulate an image before it is drawn to the screen. Shaders allow various kinds of rendering from adding and cartoony view to X-Ray outlines to rendering output. There are different Types of Shaders: Pixel Shader(DirectX)/Fragment Shader(OpenGL) where colors of individual pixels are calculated, Vertex Shader where vertex position is modified, Geometry Shader where vertices can be modified resulting in creation of geometry.

Chip8 Emulator

What exactly is an emulator? An emulator is a type of interpreter that reads the original instructions written in machine code and compiled for the target computer, interprets them, and then replaces the functionality of the target machine on the host machine.

Chip-8 was created by RCA engineer Joe Weisbecker in 1970 COSMAC VIP microcomputer. It was intended as a simpler way to mae small programs and games for the computer.

There are certain components that need to be understood before understanding how chip-8 works. -> 16 8-Bit Registers: All the operation on a CPU is done using registers.

-> 40896 Bytes of Memory As registor space is small we need memory. This space is segemented into three sections: : some reserved for CHIP-8 interpreter. : some reserved for 16 built-in characters( 0 - F) : some memory where instruction from ROM will be stored at 0x200, and after that remaing space will be free to use.

->16-bit Index Register Special register used to store memory addressed for use in operations. why 16? because it used 0xFFF which is too big for 8-bit register.

-> 16-bit Program Counter It keeps track of which instruction to execute next.

-> 16-Level stack It keeps track of order of execution when it calls into function.It can hold 16 different Program counter.

-> 8-bit stack pointer It is used to keep track of where in memory the CPU is executing. It tells us in where in the 16-levels of stack out most recent value was placed.

->8-bit DelayTimer: Timer used for timing.If there is a value it will decrement at a rate of 60Hz else it will stay zero.

-> 8-bit Sound Timer -> Input Keys

-> Display Memory It has additional memory buffer used for storing the graphics to display. Which is 64 pixels wide and 32 pixels high. Each pixel is either on or off. So, Only two color can be represented.
